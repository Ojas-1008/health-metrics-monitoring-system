Health Metrics Monitoring System - Week 1 Detailed Roadmap
Project Overview Summary
A full-stack health tracking application integrating React, Node.js, MongoDB Atlas, Apache Spark analytics, and Google Fit API for real-time health metrics monitoring and predictive analysis.[1]

Task 2.2: Database Schema Design
    • Sub-task 2.2.1: Design User Schema structure
        ◦ Fields to include:
            ▪ _id: ObjectId (auto-generated)
            ▪ email: String, unique, required
            ▪ password: String, hashed, required
            ▪ name: String, required
            ▪ googleId: String, optional (for OAuth)
            ▪ profilePicture: String (URL), optional
            ▪ googleFitConnected: Boolean, default false
            ▪ goals: Object (weight goal, step goal, sleep goal)
            ▪ createdAt: Date, default Date.now
            ▪ updatedAt: Date, auto-update
        ◦ Validations: Email format, password min length 8
        ◦ Indexes: email (unique)
    • Sub-task 2.2.2: Design Health Metrics Schema structure[1]
        ◦ Fields to include:
            ▪ _id: ObjectId
            ▪ userId: ObjectId, reference to User, required
            ▪ date: Date, required
            ▪ metrics: Object containing:
                • steps: Number, default 0
                • calories: Number, default 0
                • weight: Number, optional
                • heartRate: Number, optional
                • sleepHours: Number, optional
                • activeMinutes: Number, default 0
            ▪ source: String, enum ['manual', 'google_fit']
            ▪ createdAt: Date
            ▪ updatedAt: Date
        ◦ Indexes: userId + date (compound, unique)
    • Sub-task 2.2.3: Design Analytics Schema structure
        ◦ Fields to include:
            ▪ _id: ObjectId
            ▪ userId: ObjectId, reference to User
            ▪ analysisType: String, enum ['moving_average', 'trend', 'prediction']
            ▪ metricType: String, enum ['weight', 'steps', 'calories', 'sleep']
            ▪ timeRange: String, enum ['7days', '14days', '30days']
            ▪ result: Object (flexible structure for Spark results)
            ▪ calculatedAt: Date
            ▪ expiresAt: Date (for cache invalidation)
        ◦ Indexes: userId + analysisType + metricType (compound)
    • Sub-task 2.2.4: Design Alerts Schema structure
        ◦ Fields to include:
            ▪ _id: ObjectId
            ▪ userId: ObjectId, reference to User
            ▪ alertType: String, enum ['low_activity', 'sleep_warning', 'goal_reminder']
            ▪ message: String
            ▪ severity: String, enum ['info', 'warning', 'critical']
            ▪ read: Boolean, default false
            ▪ createdAt: Date
        ◦ Indexes: userId, read status
